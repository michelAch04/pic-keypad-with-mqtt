WiFi Credentials:

cconst char *mqtt_broker = "broker.emqx.io";

Connection Process:

WiFi connects first using WiFi.begin(ssid, password)
After WiFi connection, MQTT client connects with a unique client ID based on MAC address: "esp8266-client-" + WiFi.macAddress()
Publishes initial "hello emqx" message to the topic
Subscribes to topic "lab_micro_usek"

4. MQTT Message Format from ESP8266
From B4A to ESP8266 (Commands):
The ESP8266 receives text-based MQTT messages and converts them to serial characters:

"on1" → Serial.write(2)
"off1" → Serial.write(3)
"on2" → Serial.write(4)
"off2" → Serial.write(5)
"on3" → Serial.write('6')
"off3" → Serial.write('7')
"on4" → Serial.write('8')
"off4" → Serial.write('9')
"dc0" → Serial.write('A')
"dc33" → Serial.write('B')
"dc66" → Serial.write('C')
"dc99" → Serial.write('D')

From PIC to ESP8266 (Sensor Data):
c// ESP8266 reads from PIC via Serial and publishes
ch = Serial.read();
str = String(ch + 0);  // Bug workaround in Arduino
client.publish(topic, &str[0]);
The sensor value is sent as a single character (8-bit ADC value converted to string).
5. B4A Application MQTT Connection
Connection Setup:
basicPrivate mqtt1 As MqttClient
Private serverURI As String = "tcp://broker.emqx.io:1883"

Sub Activity_Create(FirstTime As Boolean)
    If FirstTime Then
        mqtt1.Initialize("mqtt1", serverURI, Rnd(0, 999999) & " ")
        Dim mo As MqttConnectOptions
        mo.Initialize("wifi name", "password")
        mqtt1.Connect2(mo)
    End If
End Sub
Subscription:
basicSub mqtt1_Connected (Success As Boolean)
    If Success = False Then
        Log(LastException)
        Label1.Text = "Error connecting"
    Else
        Label1.Text = "Connected"
        mqtt1.Subscribe("lab_micro_usek", 0)
    End If
End Sub
Publishing Commands:
basicmqtt1.Publish("lab_micro_usek", "on1".GetBytes("utf8"))
Receiving Data:
basicPrivate Sub mqtt1_MessageArrived (Topic As String, Payload() As Byte)
    str = BytesToString(Payload, 0, Payload.Length, "utf8")
    Label1.Text = "Your temperature is: " & str & "°C"
End Sub

7. B4A MQTT Libraries
Required Library:

jMQTT library

Enable in: Windows → Libraries Manager → check "jMQTT"

8. Additional Configuration Details
Serial Communication:

Baud Rate: 9600 baud (both PIC and ESP8266)
Format: 8 data bits, No parity, 1 stop bit (8N1)
UART Settings on PIC: SPBRG=25 for 4MHz oscillator

Hardware Setup:

PIC16F877A: 4 MHz crystal oscillator
Configuration Word: __CONFIG(0x1f72);
ESP8266: WeMos D1 Mini board
LED Pin: GPIO2 (D4) on ESP8266

Proteus Simulation:

Use COMPIM component for serial communication

Physical Port: COM1 at 9600 baud
Virtual Port: COM2 at 9600 baud (assigned to ESP8266's COM port)


The ESP8266's actual COM port number must be noted after uploading code and configured in Proteus COMPIM

Power Supply:

PIC operates at 5V
ESP8266 operates at 3.3V (built-in regulator on WeMos D1 Mini)

This setup creates a complete IoT system where the Android app controls the PIC microcontroller via MQTT over WiFi, with the ESP8266 acting as a WiFi-to-serial bridge.
